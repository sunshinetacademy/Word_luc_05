لماذا أدوات المراجعة ضرورية للحقوقي؟

تأسيس الدافع — ٥ دقايق

طالب الحقوق بيكتب بحوثاً طويلة، مذكرات، وعقوداً — كلها محتاجة مراجعة دقيقة قبل التسليم. كمان بيشتغل مع زملاء ومشرفين بيعدّلوا على نفس الملف. الأدوات دي هي الفرق بين عمل احترافي وعمل عشوائي.

سيناريو حقيقي الدكتور المشرف رجّع البحث بـ ١٥ تعليق وتعديل. بدون Track Changes — مش هتعرفي إيه اللي اتغيّر. بدون Comments — مش هتفهمي السبب. بدون Find & Replace — هتقعدي تدوري على اسم مكرر في ٣٠ صفحة يدوياً.

ج. Find & Replace المتقدم (More Options)

Ctrl + H → More >>
الخيار المتقدمما يفعلهالاستخدام في الحقوق
Match Case تطابق حالة الأحرف (كبيرة/صغيرة) البحث عن "القانون" بدون "القانوني" أو "قانون"
Find Whole Words Only البحث عن الكلمة كاملة فقط البحث عن "عقد" بدون "عقدة" أو "عقود"
Use Wildcards استخدام رموز بحث خاصة (مثل * و ?) البحث عن "م. ١*" لإيجاد كل المواد القانونية
Find All Word Forms البحث عن كل تصريفات الكلمة البحث عن "حكم" يشمل "يحكم" و"الحكم"
Format → Font البحث عن نص بتنسيق محدد إيجاد كل النصوص Bold لتوحيد تنسيقها
Special → Paragraph Mark البحث عن رموز خاصة مثل فقرات أو tabs حذف الأسطر الفارغة الزيادة في المستند كله
📋 مثال ١ — تغيير اسم موكّل في عقد كامل

السيناريو: عقد بيع من ٢٠ صفحة — اكتشفتِ أن اسم المشتري مكتوب غلط في كل العقد

  1. اضغطي Ctrl + H لفتح Find & Replace
  2. في "Find what": اكتبي الاسم الخاطئ مثلاً "محمد أحمد علي محمود"
  3. في "Replace with": اكتبي الاسم الصحيح "محمد أحمد علي محمد"
  4. اضغطي Replace All
  5. Word سيعرض: "تم استبدال ١٨ حالة" — الاسم تغيّر في كل العقد في ثانية واحدة
  6. اضغطي Ctrl + Z لو أردتِ التراجع
توفير الوقت بدون Replace All كنتِ ستحتاجين لتغيير الاسم يدوياً ١٨ مرة — مع احتمال نسيان بعضها!
📋 مثال ٢ — حذف الأسطر الفارغة الزيادة من بحث

السيناريو: بحث فيه أسطر فارغة متعددة بين الفقرات — يجعله يبدو غير منظم

  1. اضغطي Ctrl + H
  2. اضغطي More >> ثم Special
  3. في "Find what": اختاري Paragraph Mark مرتين — ستظهر "^p^p"
  4. في "Replace with": اختاري Paragraph Mark مرة واحدة — "^p"
  5. اضغطي Replace All — كرّري حتى يقول "تم استبدال ٠ حالة"
تنبيه الأسلوب ده بيحذف كل الأسطر الفارغة المزدوجة. لو البحث محتاج مسافة بين الفصول — استخدمي Page Break بدلاً من أسطر فارغة من البداية.
تدريب ١ — Find & Replace في بحث قانوني

على ملف بحث من المحاضرات السابقة:

  1. استخدمي Ctrl + F لإيجاد كلمة "القانون" — كم مرة تكررت؟
  2. استخدمي Ctrl + H لاستبدال "محكمة النقض" بـ "محكمة النقض المصرية" في كل المستند
  3. جرّبي More Options → Match Case وابحثي عن "قانون" (بالحرف الصغير) وشاهدي النتيجة
  4. استبدلي كل النص Bold بنص Normal عبر: Format → Font → Bold (في Find) ← اتركي Replace فارغة
  5. تراجعي عن كل التغييرات بـ Ctrl + Z

03

تتبع التغييرات (Track Changes)

الأداة الأهم للعمل مع المشرف — ٢٠ دقيقة
Track Changes أداة تُسجّل كل تعديل يحدث في المستند — إضافة نص، حذف نص، تغيير تنسيق — مع اسم المعدِّل والتاريخ. كل تغيير يظهر بلون مختلف ويمكن قبوله أو رفضه لاحقاً.
Review → Track Changes → Track Changes

أو الاختصار: Ctrl + Shift + E (تفعيل/إيقاف)

أ. كيف يبدو المستند مع Track Changes مفعّل

تُعدّ المسؤولية المدنية التقصيرية العقدية من أهم موضوعات القانون المدني المصري.

وقد نظّم المشرّع المصري أحكامها في المواد من ١٦٣ إلى ١٧٨ من القانون المدني.

تعليق: يرجى إضافة مرجع هناوقد استقر الفقه على أن هذه المسؤولية تقوم على ثلاثة أركان.

نص أخضر مسطّر = إضافة جديدة نص أحمر مشطوب = نص محذوف مستطيل ذهبي = تعليق

ب. قبول أو رفض التغييرات

Review → Accept / Reject

Accept This Change

قبول التغيير الحالي فقط — التعديل يصبح جزءاً ثابتاً من النص.

Accept All Changes

قبول كل التغييرات دفعة واحدة — يُنظّف المستند من كل العلامات.

Reject This Change

رفض التغيير والعودة للنص الأصلي قبل التعديل.

Reject All Changes

إلغاء كل التغييرات والعودة للنسخة الأصلية كاملة.

ج. إعدادات العرض (Show Markup)

Review → Show Markup

All Markup

عرض كل التغييرات والتعليقات بالألوان. الوضع الافتراضي عند المراجعة.

Simple Markup

يُظهر علامة حمراء في الهامش فقط بدون تفاصيل. لقراءة أوضح.

No Markup

يُخفي كل التغييرات ويعرض الشكل النهائي كما سيبدو بعد القبول.

سر احترافي قبل إرسال البحث للمشرف — فعّلي Track Changes أولاً ثم اطلبي منه يعدّل. هكذا ترين بالضبط ما غيّره دون أن تضطري لمقارنة نسختين يدوياً.
📋 مثال ٣ — مراجعة بحث مع المشرف

السيناريو: أرسلتِ بحث "عقد الإيجار" للدكتور — رجع بتعديلات كثيرة

  1. افتحي الملف الذي أرسله الدكتور — التعديلات ستظهر بألوان تلقائياً
  2. اذهبي إلى: Review → Show Markup → All Markup
  3. اقرئي كل تعديل — انتقلي بين التعليقات بـ: Review → Next Comment
  4. لكل تعليق: إما عدّلي النص ثم احذفي التعليق، أو اتركيه مؤقتاً
  5. للتعديلات النصية: قرّري قبول أو رفض كل تعديل
  6. في النهاية: Accept All Changes → احفظي نسخة جديدة باسم "بحث_الإيجار_نسخة٢"
تدريب ٢ — Track Changes بين الطالب والمشرف

شغّلي Track Changes وطبّقي التعديلات التالية:

  1. فعّلي Track Changes: Ctrl + Shift + E
  2. اكتبي جملة جديدة في نهاية أي فقرة — ستظهر بلون مختلف
  3. احذفي كلمة من وسط نص — ستظهر مشطوبة بدل ما تختفي
  4. غيّري تنسيق كلمة (Bold) — ستظهر علامة في الهامش
  5. أوقفي Track Changes: Ctrl + Shift + E
  6. اقبلي تعديلاً واحداً (Accept) وارفضي آخر (Reject) وشاهدي النتيجة
  7. اضغطي Accept All Changes وشاهدي المستند نظيفاً

04

التعليقات (Comments)

للتواصل بدون تغيير النص — ١٠ دقايق
Comment (التعليق) ملاحظة نصية مرتبطة بجزء محدد من المستند — تظهر في الهامش. لا تغيّر النص الأصلي. يمكن الرد عليها وحذفها.

أ. إضافة تعليق جديد

حدّدي النص المراد التعليق عليه → Review → New Comment

أو: Ctrl + Alt + M

ب. التنقل بين التعليقات

Next / Previous Comment

Review → Next/Previous للتنقل بين التعليقات بالترتيب في المستند الطويل.

Delete Comment

يمين الماوس على التعليق → Delete Comment | أو Review → Delete All Comments.

ج. الرد على تعليق (Reply to Comment)

في Word 365، يمكن الرد على تعليق مباشرةً داخله — مثل محادثة. مفيد جداً في المراجعة المشتركة مع الدكتور أو الزملاء.

الفرق بين Comment و Track Changes Track Changes = تعديل فعلي في النص يمكن قبوله أو رفضه. Comment = ملاحظة جانبية للشرح أو التساؤل — لا تغيّر النص وتُحذف يدوياً. في الحقوق: المشرف يستخدم التعليقات ليقول "أضيفي مرجعاً هنا" دون تعديل النص نفسه.

د. طباعة المستند مع أو بدون التعليقات

File → Print → Print All Pages → Print Markup (لطباعة التعليقات)

أو اختاري Document بدلاً من Document Showing Markup لطباعة بدون تعليقات.

05

التدقيق الإملائي والنحوي

الخط الدفاعي الأخير — ١٠ دقايق

وثيقة قانونية فيها خطأ إملائي = وثيقة غير محترمة. Word بيوفّر أدوات تدقيق قوية لكن لها حدود — يجب فهمها جيداً.

أ. التدقيق التلقائي أثناء الكتابة

تُعدّ المسؤلية المدنية من أهم أحكام القانون المدني المصرى.

خط أحمر متعرج = خطأ إملائي خط أزرق متعرج = خطأ نحوي أو أسلوبي

ب. تشغيل التدقيق الكامل للمستند

Review → Spelling & Grammar

أو الاختصار: F7

الخيارما يفعله
Ignore Onceتجاهل الخطأ الحالي مرة واحدة فقط
Ignore Allتجاهل كل مواضع هذه الكلمة في المستند
Add to Dictionaryإضافة الكلمة للقاموس — لن تُعلَّم كخطأ مستقبلاً
Changeتطبيق التصحيح المقترح على الخطأ الحالي
Change Allتطبيق التصحيح على كل مواضع الكلمة
AutoCorrectإضافة الكلمة للتصحيح التلقائي مستقبلاً
حدود التدقيق الإملائي في العربية التدقيق الإملائي العربي في Word ليس مثالياً. كثير من المصطلحات القانونية العربية قد تُعلَّم كأخطاء مثل "الإيجاب" أو "المُعسِر". الحل: أضيفيها للقاموس (Add to Dictionary) من أول مرة.

ج. Thesaurus — قاموس المترادفات

Review → Thesaurus

أو: Shift + F7 على الكلمة المراد إيجاد مرادفاتها

مفيد في الكتابة القانونية لتجنب تكرار نفس المصطلح عشرات المرات في البحث.

د. Word Count — عد الكلمات

Review → Word Count

أو انظري لشريط الحالة في أسفل الشاشة — يعرض عدد الكلمات دائماً.

ضروري عندما الأستاذ يحدد حداً أقصى لعدد الكلمات في البحث.

06

حماية المستند (Document Protection)

للعقود والوثائق الرسمية — ١٠ دقايق

في الحقوق، بعض الوثائق لا يجب أن يعدّل عليها أحد بعد التوقيع أو الاعتماد. Word يوفر عدة مستويات من الحماية.

أ. Restrict Editing — تقييد التعديل

Review → Restrict Editing
📝

Formatting Restrictions

منع تغيير التنسيق مع السماح بالكتابة. مفيد لنماذج العقود الموحّدة.

🔒

Editing Restrictions

خيارات: No Changes (القراءة فقط) | Comments فقط | Track Changes فقط | Filling in Forms.

ب. Protect with Password — حماية بكلمة مرور

File → Info → Protect Document → Encrypt with Password
تحذير مهم لو نسيتِ كلمة المرور — لا توجد طريقة رسمية لاسترجاعها في Word. دوّنيها في مكان آمن دائماً.

ج. Mark as Final — تعليم بالنهائي

File → Info → Protect Document → Mark as Final

يحوّل المستند لوضع القراءة فقط مع رسالة تنبيه. لكنه ليس حماية حقيقية — أي شخص يستطيع إلغاءه بضغطة. يُستخدم للإشارة فقط.

📋 مثال ٤ — تأمين نموذج عقد بحيث لا يُعدَّل إلا في الخانات المحددة

السيناريو: أعددتِ نموذج عقد إيجار — تريدين إرساله للموكل ليملأ بياناته فقط دون تعديل البنود

  1. Review → Restrict Editing
  2. فعّلي: ☑ Allow only this type of editing: Filling in forms
  3. في مكان البيانات (الاسم، التاريخ، المبلغ): أدرجي حقول نماذج (Developer → Text Content Control)
  4. اضغطي Yes, Start Enforcing Protection وأدخلي كلمة مرور
  5. الآن: الموكل يستطيع فقط ملء الخانات — لا يستطيع تغيير بنود العقد
07

التصحيح التلقائي والاختصارات الذكية

توفير الوقت في الكتابة المتكررة — ٨ دقايق

أ. AutoCorrect — التصحيح التلقائي

File → Options → Proofing → AutoCorrect Options
AutoCorrect ميزة تُصحّح تلقائياً الأخطاء الشائعة فور كتابتها. يمكن إضافة اختصارات مخصصة مثل: كتابة "ق.م" يتحول تلقائياً إلى "القانون المدني المصري".

استخدامات AutoCorrect في الحقوق

الاختصار المكتوبالنص الكامل التلقائيالفائدة
ق.مالقانون المدني المصرييُكتب في كل بحث عشرات المرات
ق.عقانون العقوبات المصريتوفير في الكتابة المتكررة
م.نمحكمة النقض المصريةالمرجع الأكثر تكراراً في البحوث
م.االمحكمة الإدارية العليابحوث القانون الإداري
ت.جالتقنين التجاري المصريبحوث القانون التجاري

ب. Building Blocks — كتل النصوص الجاهزة

Insert → Quick Parts → AutoText
Building Blocks / AutoText حفظ أي نص أو جزء من مستند واستدعاؤه في أي وقت بكلمة مفتاحية. مثالي لحفظ الديباجة القانونية، بنود العقود المتكررة، أو صيغ المذكرات الثابتة.

كيفية إنشاء AutoText: حدّدي النص ← Insert → Quick Parts → Save Selection to Quick Part Gallery

كيفية استدعائه: اكتبي الاسم المختصر ثم اضغطي F3

تدريب ٣ — إعداد AutoCorrect واختصارات قانونية

أعدّي بيئة Word مخصصة للكتابة القانونية:

  1. اذهبي إلى: File → Options → Proofing → AutoCorrect Options
  2. أضيفي اختصاراً: في "Replace" اكتبي "قم" وفي "With" اكتبي "القانون المدني المصري"
  3. اضغطي Add ثم OK — جرّبيه في مستند جديد
  4. اكتبي ديباجة عقد كاملة (٣-٤ أسطر): "بسم الله الرحمن الرحيم، عقد بيع مبرم بين..."
  5. حدّدي النص: Insert → Quick Parts → Save Selection to Quick Part Gallery — اسميه "ديباجة عقد"
  6. في مستند جديد: اكتبي "ديباجة عقد" واضغطي F3 — ستظهر الديباجة كاملة

أداة احترافية للمراجعة — ٥ دقايق
Compare Documents أداة تقارن نسختين من نفس المستند وتعرض الفروق بينهما تلقائياً كـ Track Changes. مفيد عندما يُعدَّل ملف خارج Track Changes.
Review → Compare → Compare
📄

Original Document

النسخة الأصلية (المرسلة للمشرف مثلاً).

📝

Revised Document

النسخة المعدّلة (التي أرجعها المشرف بعد التعديل).

متى تستخدمينه في الحقوق أرسلتِ عقداً للمحامي الآخر — رجع بنسخة "معدّلة" لكن بدون Track Changes. استخدمي Compare لمعرفة كل ما تغيّر في العقد بدقة — حرفاً حرفاً.

قاموس مصطلحات المحاضرة

مرجع سريع للمراجعة
Find & Replace
البحث والاستبدال — أداة لإيجاد نص في المستند واستبداله تلقائياً في كل الأماكن دفعة واحدة.
Wildcards
رموز بحث خاصة مثل * و ? تُستخدم في Find المتقدم للبحث عن أنماط نصية غير محددة تماماً.
Track Changes
تتبع التغييرات — يُسجّل كل تعديل في المستند بلون مميز مع اسم المعدِّل والتاريخ.
Accept / Reject
قبول أو رفض التغييرات المُتتبَّعة. Accept يثبّت التعديل، Reject يُعيد النص الأصلي.
Show Markup
خيارات عرض التغييرات والتعليقات: All Markup (كل شيء) / Simple Markup (علامات فقط) / No Markup (الشكل النهائي).
Comment
تعليق جانبي مرتبط بنص محدد — لا يغيّر المحتوى ويمكن الرد عليه وحذفه.
Spelling & Grammar
التدقيق الإملائي والنحوي — يفحص المستند ويقترح تصحيحات. الاختصار: F7.
Add to Dictionary
إضافة كلمة للقاموس الشخصي حتى لا تُعلَّم كخطأ مستقبلاً.
Thesaurus
قاموس المترادفات — يقترح كلمات بديلة لأي كلمة محددة. الاختصار: Shift + F7.
Restrict Editing
تقييد التعديل — يمنع تغيير المستند أو يحصره في نوع محدد (تعليقات فقط / نماذج فقط).
AutoCorrect
التصحيح التلقائي — يصحح الأخطاء الشائعة فور كتابتها ويتيح إنشاء اختصارات نصية مخصصة.
Building Blocks / AutoText
كتل نصية جاهزة محفوظة يمكن استدعاؤها فوراً بكلمة مفتاحية + F3.
Compare Documents
مقارنة نسختين من مستند وعرض الفروق كـ Track Changes — حتى لو لم يُفعَّل Track Changes أثناء التعديل.
Encrypt with Password
تشفير المستند بكلمة مرور — لا يمكن فتحه بدونها.
Mark as Final
تعليم المستند كنسخة نهائية للقراءة فقط — إشارة غير ملزمة يمكن إلغاؤها.

التدريب الختامي الشامل

تجميع كل المحاضرة في سيناريو قانوني واقعي — ١٢ دقيقة
تدريب ختامي — مراجعة عقد إيجار كاملة

السيناريو: وصلك عقد إيجار من الطرف الآخر بتعديلات غير معلّمة — راجعيه احترافياً

  1. Compare: Review → Compare → اختاري النسخة الأصلية والنسخة المعدّلة — شاهدي الفروق تلقائياً
  2. Find & Replace: ابحثي عن "المستأجر" وتأكدي أن الاسم صحيح في كل المستند — Ctrl + H
  3. التدقيق: شغّلي F7 وصحّحي كل الأخطاء الإملائية — أضيفي المصطلحات القانونية للقاموس
  4. Track Changes: فعّلي Ctrl + Shift + E وعدّلي بند الإيجار — اكتبي مبلغاً جديداً
  5. Comment: أضيفي تعليقاً على بند الفسخ: "يحتاج مراجعة محامٍ" — Ctrl + Alt + M
  6. AutoText: احفظي ديباجة العقد كـ Building Block باسم "ديباجة إيجار"
  7. الحماية: Review → Restrict Editing → Comments Only → Enforce Protection بكلمة مرور
  8. الحفظ: احفظي باسم "عقد_الإيجار_مراجعة_اسمك" — Ctrl + Shift + S (Save As)
⏱ الوقت المتوقع: 12 دقيقة

الواجب البيتي والمراجعة الذاتية

سلّميه في المحاضرة القادمة

الواجب: مراجعة بحث المحاضرات السابقة احترافياً

  • Find & Replace: ابحثي عن اسم القانون الرئيسي في بحثك (مثل "القانون المدني") وتأكدي من كتابته بشكل موحّد في كل البحث — صحّحي أي تباين في الكتابة.
  • التدقيق الإملائي: شغّلي F7 على كل البحث — صحّحي الأخطاء وأضيفي المصطلحات القانونية للقاموس. سجّلي كم خطأً وجدتِ.
  • Track Changes: فعّلي Track Changes وأجري ٣ تعديلات على البحث (إضافة، حذف، تعديل) — ثم احفظي الملف. لا تقبلي التعديلات بعد.
  • Comments: أضيفي ٣ تعليقات على أجزاء من البحث (مثل: "يحتاج مرجع"، "راجع الصياغة"، "إضافة مثال").
  • AutoCorrect: أعدّي ٣ اختصارات مخصصة لأكثر مصطلحات بحثك تكراراً.
  • الحماية: احفظي نسخة من البحث بكلمة مرور (Encrypt with Password) — وسلّميها مع ذكر كلمة المرور.
